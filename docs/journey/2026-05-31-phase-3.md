# Phase 3 — Next.js Dashboard

## Goal
Build a web interface so anyone can see brand visibility data without touching the terminal. The dashboard fetches from the FastAPI server and displays charts, scores, and Claude's plain-language insights.

## What I Built
3 pages:
- `/` — brand list with cards linking to each brand
- `/brands/[id]` — brand detail: visibility score, over-time chart, all past insights
- Audit button that triggers a background job and polls for completion

4 files:
- `app/page.tsx` — brand list (server component)
- `app/brands/[id]/page.tsx` — brand detail (server component)
- `app/brands/[id]/AuditButton.tsx` — audit trigger + polling (client component)
- `app/brands/[id]/VisibilityChart.tsx` — bar chart (client component)

## What I Learned About Next.js

**Server vs Client components:**
Server components run on the server — they can fetch data directly but can't handle clicks or state. Client components run in the browser — they can use useState, handle events, use browser APIs. Default is server. Add `"use client"` at the top to make it a client component.

**App Router folder structure:**
Files at `app/brands/[id]/page.tsx` automatically become the page for `/brands/1`, `/brands/2`, etc. The `[id]` in brackets is a dynamic segment — Next.js captures whatever is in that URL position.

**Why charts must be client components:**
Recharts uses React's createContext and renders SVG — both browser-only. Importing it in a server component causes `createContext is not a function`. Fix: move chart code to its own file with `"use client"` at the top.

## AI vs Non-AI
Zero LLM usage in the dashboard. It only reads data that was already generated and saved during the audit. The journalist (Claude) writes once; the newspaper (dashboard) serves it to readers indefinitely.

## Bugs I Hit

**Bug 1 — 404 on /brands/1**
Cause: In Next.js 15+, route params are now a Promise and must be awaited. `params.id` without await returns undefined. `Number(undefined) = NaN`. No brand matched. `notFound()` fired.
Fix: Change `{ params: { id: string } }` to `{ params: Promise<{ id: string }> }` and add `const { id } = await params`.
Lesson: Next.js introduces breaking changes between major versions. Always check the migration guide when upgrading.

**Bug 2 — createContext is not a function**
Cause: Recharts imported directly in a server component.
Fix: Moved recharts code to `VisibilityChart.tsx` with `"use client"`.
Lesson: Any library that uses React hooks or browser APIs must live in a client component.

**Bug 3 — Chart bars not visible in Playwright screenshot**
Cause: Recharts calculates chart dimensions from the container size. Playwright's headless browser renders at 0 height for ResponsiveContainer. Works fine in a real browser.
Not a code bug — a headless rendering limitation.

## What I Learned
- Two servers must run simultaneously: FastAPI on 8000, Next.js on 3000. One terminal tab each.
- `fetch()` in server components runs at request time. `{ cache: "no-store" }` ensures fresh data every request.
- The audit button polls every 4 seconds — simple interval, no WebSockets needed.
- npm dependency conflicts (`--legacy-peer-deps`) are common when mixing packages with different peer dependency expectations. Usually safe for dev tooling conflicts.
