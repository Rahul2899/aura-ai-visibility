# Phase 2 — FastAPI Wrapper

## Goal
Wrap the Python pipeline in an HTTP API so the dashboard (Phase 3) can talk to it. A browser can't run Python commands — it can only send HTTP requests.

## What I Built
A REST API with 6 endpoints:
- GET /brands — list all brands
- POST /brands — create a brand
- GET /brands/{id}/report — visibility scores (reads from runs table)
- GET /brands/{id}/insights — Claude's audit summaries (reads from insights table)
- POST /audit/brands/{id} — start an orchestrated audit in background, returns job_id immediately
- GET /audit/{job_id} — poll audit progress

## What I Learned About Servers
A server is a process that runs continuously, waiting for requests. `uvicorn` is the program that runs the FastAPI app. `--port 8000` means listen on port 8000. Ports separate multiple programs using the same internet connection (Postgres = 5432, FastAPI = 8000, Next.js = 3000).

`localhost` and `127.0.0.1` mean the same thing — this machine only, not the internet.

`curl` is a terminal tool that sends HTTP requests and prints responses. Same as what a browser does, without the visual layer.

## AI vs Non-AI Decisions
Every endpoint is pure data retrieval — SQL queries and arithmetic. No LLM anywhere in the API layer. The LLM work already happened during the audit and was saved to the database. The API just reads it back out.

## The Background Job Pattern
An orchestrated audit takes 3-5 minutes. HTTP connections are supposed to complete in under a second. Solution: POST /audit/brands/1 starts the job and returns {"job_id": "job_1"} immediately. The audit runs in the background. The frontend polls GET /audit/job_1 every few seconds until status = "completed".

In-memory dict for job state — not the database — because jobs are ephemeral. If the server restarts, jobs are gone anyway.

## Bugs / Confusions

**404 on first try**
Cause: Opened http://localhost:8000/ in browser — the root path has no route.
Fix: Use the full path http://localhost:8000/brands
Lesson: 404 = route not found, not "server broken". Always check the exact URL.

**GET /favicon.ico 404**
Not a bug. Every browser automatically requests a favicon (tab icon) for any URL you visit. FastAPI doesn't serve one. Harmless.

**report endpoint showed 0.0% visibility**
Cause: The report endpoint reads from the `runs` table (old batch pipeline). The orchestrator writes to `insights` table. Two separate data paths.
Fix: Use the /insights endpoint to see orchestrator results.
Lesson: Know which table each feature writes to before debugging why numbers look wrong.

## What I Learned
- HTTP is just a request/response protocol. GET = read, POST = write/trigger.
- FastAPI maps URLs to Python functions automatically. You write the function, decorate it with @router.get("/path"), done.
- The docs page at /docs is generated automatically by FastAPI — free interactive testing UI with no extra code.
- Separating routes into different files (brands.py, audits.py) keeps things manageable as the API grows.
