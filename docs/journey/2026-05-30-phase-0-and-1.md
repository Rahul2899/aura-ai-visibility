# Phase 0 + 1 — Foundation & Orchestrator

## Goal
Fix the broken foundation (no migrations, missing eval data) and build the core of the project: an AI agent that autonomously audits brand visibility across 6 LLM models.

## What I Built
An agentic loop where Claude Haiku (via AWS Bedrock) decides what probe questions to ask about a brand, runs them across 6 AI models concurrently, and synthesizes the results into a plain-language insight.

The system:
- Generates its own probe questions (no hardcoded prompts)
- Runs 6 models in parallel per probe (asyncio.gather + semaphore)
- Tracks probe performance over time (hit_rate = arithmetic, no LLM)
- Saves Claude's findings to the `insights` table

## AI vs Non-AI Decisions
- Generating probe questions: **LLM** — context-dependent, no fixed grammar
- Running probes against models: **not LLM** — deterministic dispatch
- Checking if brand is mentioned: **not LLM** — simple string match (target.lower() in response.lower())
- Updating probe hit rate: **not LLM** — hit_count / run_count arithmetic
- Synthesizing findings: **LLM** — requires language reasoning

## Bugs I Hit

**Bug 1 — `ModuleNotFoundError: asyncpg`**
Cause: Used system Python without running `make install` first.
Fix: `pip install asyncpg`
Lesson: Always `make install` first. requirements.txt is the contract.

**Bug 2 — `Invalid type for messages[0].content, type: str, valid types: list`**
Cause: OpenAI format uses `"content": "string"`. Bedrock Converse API requires `"content": [{"text": "string"}]`. I used OpenAI format by mistake.
Fix: Wrap content in `[{"text": ...}]`
Lesson: Different APIs have different message formats. Bedrock uses content blocks (list), OpenAI uses plain strings.

**Bug 3 — `ValidationException: model identifier is invalid`**
Cause: Used `anthropic.claude-haiku-4-5` — missing date and version suffix.
Correct: `us.anthropic.claude-haiku-4-5-20251001-v1:0`
The `us.` prefix = cross-region inference (AWS routes to available datacenter). Required for newer models.
Lesson: Bedrock model IDs are exact strings with date + version. Verify from the Bedrock console.

**Bug 4 — `404 Not Found` for Gemini and DeepSeek on OpenRouter**
Cause: Free model IDs expire. `google/gemini-2.0-flash-exp:free` and `deepseek/deepseek-chat-v3-0324:free` were retired without notice.
Fix: Checked openrouter.ai/models, updated to `google/gemma-4-31b-it:free` and `openai/gpt-oss-120b:free`
Lesson: Free API model IDs are unstable. In production, store them in DB/config — not hardcoded.

**Bug 5 — Rate limiting on Llama (OpenRouter free tier)**
Cause: `meta-llama/llama-3.3-70b-instruct:free` is overloaded. Free tier throttles aggressively.
Fix: Swapped for `nvidia/nemotron-super-49b-v1:free`
Lesson: Free tiers are unreliable. This is why the project has both OpenRouter (free/dev) and Bedrock (paid/production).

Claude generated this analysis autonomously — I didn't write any analysis code.

## What I Learned
- Agentic loops are just a while loop + tool dispatch. The "intelligence" is entirely in the LLM deciding which tools to call.
- AWS Bedrock and OpenAI have different API conventions even for the same concept (message format). Switching providers means reading new docs.
- Model bias is real and measurable. Different LLMs trained on different data give systematically different answers about the same brand.
- The most useful findings (feature-category gap, model bias) emerged from the data automatically — not from code I wrote.
